# ADR 0006 — Attendance에 status 추가 (참가/안함)

- 상태: 채택(Accepted)
- 날짜: 2026-07-13
- 관련: [ADR 0004](0004-binary-attendance-date-only.md)의 결정 1(이진 Attendance)을 개정

## 맥락

이진 모델(행 존재 = 참가 / 없음 = 무반응)에선 침묵이 "못 봤음"인지 "안 옴"인지 구분 불가 → 띄운 사람이 모임 성사를 판단하기 어렵다는 피드백. 반면 명시적 "불참"을 너무 드러내면 무반응자에게 압박이 생겨 "부담 없이 무시" 가치가 훼손됨.

## 결정

1. **`Attendance.status` enum 추가**: `GOING`(참가) | `NOT_GOING`(안함), 기본 `GOING`. 행 없음 = 무반응은 유지 — 3상태(참가/안함/무반응).
2. **"안함"은 조용하게 표시**: 아바타는 참가자만. 안함은 `안함 N` 카운트 텍스트만(이름 비노출). 무반응자는 어디에도 안 보임 → 압박 최소화.
3. **UI는 두 버튼**(참가 | 안함), 같은 버튼 다시 누르면 해제(무반응 복귀). 액션은 `toggleAttendance` → `setAttendance(hangoutId, status)`로 교체.

## 결과

- 장점: 띄운 사람의 성사 판단 명확, 응답자는 한 탭으로 의사 표시. 무시 옵션은 그대로 살아 있음.
- 단점: 스키마·UI 복잡도 소폭 증가. "미정" 같은 중간상태는 여전히 제외(백로그 유지).
- 마이그레이션: 기존 행은 전부 참가였으므로 `DEFAULT 'GOING'`이 사실과 일치.
- ADR 0004의 결정 2(날짜만 저장)는 그대로 유효.
