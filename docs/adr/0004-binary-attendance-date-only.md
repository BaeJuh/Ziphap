# ADR 0004 — 이진 Attendance + 날짜만 저장 데이터 모델

- 상태: 채택(Accepted)
- 날짜: 2026-06-22

## 맥락

핵심 가치는 부담 없는 "놀자 띄우기". Doodle/when2meet식 격식 조율이 아님. 모델은 최소주의로 5개:

```
User ─(Membership)─ Group ─1:N─ Hangout ─1:N─ Attendance
```

## 결정

1. **Attendance는 이진(binary).** 행이 존재 = 참가 / 없음 = 무반응. **`status` 컬럼을 두지 않는다.** "불가" 표시나 중간상태(미정 등)는 백로그.
2. **Hangout은 날짜만 저장**(`date`). 시간은 자유 텍스트(`timeText`, 예: "퇴근후"), 한 줄 메모(`note`). 정밀 시각을 저장하지 않아 **타임존 처리를 회피**한다.

## 결과

- 장점: 스키마·UI 단순, 타임존 버그 없음, "가볍게 띄우기" 가치에 부합.
- 단점: "불가/미정" 같은 풍부한 응답 불가, 정확한 시각 정렬·알림 불가. 모두 의도적 제외(백로그).
- "이날 불가" 표시가 필요해지면 이 ADR을 대체하는 새 ADR로 status 모델 재논의.
