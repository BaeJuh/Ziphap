# ADR 0001 — 모놀리식 Next.js + Railway 올인원 호스팅

- 상태: 채택(Accepted)
- 날짜: 2026-06-22

## 맥락

소규모 캐주얼 모임 캘린더. 프론트/백을 따로 운영할 팀도, 분리해서 얻을 트래픽 규모도 없음. 간편함이 최우선.

## 결정

프론트엔드와 백엔드를 **하나의 Next.js 16 (App Router) 앱**으로 합친다(모놀리식).

- 읽기: Server Component가 서버에서 DB를 직접 조회 → HTML 렌더. 별도 API 왕복 없음.
- 쓰기: 클라이언트가 **Server Action**으로 서버 함수 호출. (필요 시에만 Route Handler)
- "백엔드가 프론트를 호출"하거나 "프론트가 분리된 백엔드를 REST로 부르는" 구조가 **아니다.** 한 프로세스 안에서 서버 쪽이 렌더링과 DB를 모두 처리한다.

호스팅은 **Railway 올인원**: 앱 컨테이너 1개 + Railway 관리형 Postgres 플러그인. 앱은 Nixpacks가 Next.js를 자동 감지해 빌드(현재 Dockerfile 불필요).

## 결과

- 장점: 배포 대상 1개, 인프라 단순, CORS·서비스 간 인증 불필요.
- 단점: 프론트/백 독립 스케일 불가(현 규모에선 무의미). 추후 분리하려면 재설계 필요.
- 관련: 로컬 개발 DB는 [0002](0002-postgres-everywhere.md) 참고.
