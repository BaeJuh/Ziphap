# ADR 0005 — 캘린더 표시 범위: 오늘부터 4주 롤링

- 상태: 채택(Accepted)
- 날짜: 2026-06-22

## 맥락

README 초안엔 "월간 캘린더(1일~말일)"로만 적혀 있었으나, 실제 표시 범위를 정하지 않았음. 집합의 "놀자 띄우기"는 **가까운 미래**(이번 주말, 다음 주)가 대부분 — 장기 플래너가 아님.

전통 월간 그리드의 문제: 오늘이 월 후반이면 그리드 위쪽이 **지난 날짜로 빈 공간**이 되고, 정작 중요한 "앞으로 며칠"이 아래로 몰림.

## 결정

캘린더는 **오늘이 포함된 주의 월요일부터 4주(28일)를 롤링** 표시한다.

- **월요일 시작**, 토=파랑 / 일=빨강 (README 디자인 유지).
- 월 경계와 무관하게 연속 표시(달 넘어가도 이어짐). 월 이동(이전/다음) 네비게이션은 **백로그** — 칩이 생긴 뒤(Slice 3 이후) 필요하면 추가.
- "오늘" 기준은 **Asia/Seoul**로 계산(사용 대상이 한국 지인). 날짜 산술은 UTC 기반으로 처리해 타임존 드리프트 방지. 날짜 저장은 date-only ([ADR 0004](0004-binary-attendance-date-only.md)).

> 4주는 기본값일 뿐 조정 가능(`buildCalendarWeeks(weeks)`).

## 결과

- 장점: 항상 "오늘부터 앞으로"가 보임, 지난 날 낭비 없음, 캐주얼 모임 성격에 부합.
- 단점: 4주 너머 먼 미래 약속은 못 띄움(현 성격엔 무방). 필요해지면 주 수 늘리거나 월 이동 추가.
- README의 "월간 캘린더" 표현은 이 ADR로 대체.
